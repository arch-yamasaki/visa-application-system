# 2026-05-01 ヒアリング要約

## 目的

在留資格申請業務をAIエージェントで支援できるかを確認するため、実際の案件流入、書類収集、資格分類、オンライン申請入力、判断が必要な箇所をヒアリングした。

## 最重要ポイント

初期の主対象は技人国に寄せる。正式名称は「技術・人文知識・国際業務」。建設会社、IT、翻訳・通訳、貿易、マーケティング、デザイン、管理系など、一般的な就労ビザ案件の多くがここに集まる。ヒアリングでは、フォースバレー経由の案件はほぼ技人国と見てよい、という話があった。

技人国では「大学・専門学校などで学んだ内容」と「日本で行う仕事内容」が対応しているかが重要。単純労働や現場作業員に見えると厳しい。活動内容詳細の書き方も重要で、過去に通った文面や理由書の蓄積が価値になる。

## 案件の入口

案件の入り方は主に次のパターン。

1. 外国人本人からの相談
2. 外国人を雇いたい会社からの相談
3. 人材紹介会社からの相談
4. 事業協同組合からの相談
5. 弁護士・士業などからの紹介

特に人材紹介会社ではフォースバレーの話が多く出た。フォースバレーは外国人材側の情報を持っており、雇用会社側の情報は会社から集める流れになる。一般の雇用会社から直接来る案件では、どの資格で申請するか、どんな仕事をさせるか、学歴・職歴が合っているかをより丁寧に聞く必要がある。

## 技人国で見るべきこと

技人国では、以下が重要。

- 仕事内容が技術・人文知識・国際業務に該当するか
- 現場作業・単純労働に見えないか
- 学歴、専攻、成績証明書の科目と業務内容がつながるか
- 職歴が業務内容に関係しているか
- 翻訳・通訳などでは日本語能力や語学力の証明があるか
- 雇用条件通知書、雇用契約書、履歴書、卒業証明書、成績証明書の内容に矛盾がないか
- 活動内容詳細の記載が審査上不利な表現になっていないか

会社ホームページだけで仕事内容を推定するのは不十分。大きい会社では部署や職務が多く、建設会社でも翻訳、貿易、施工管理、管理業務などで意味が変わる。選択肢を用意して会社側に職務を選んでもらう設計が良い。

## 書類と情報収集

主な入力資料は以下。

- 旅券、パスポート
- 在留カード
- 履歴書、職務経歴書
- 卒業証明書
- 成績証明書
- 雇用条件通知書、雇用契約書
- 会社情報、決算書、会社カテゴリーに関する資料
- 申請人の家族情報、在日親族、過去の来日歴
- 申請人の資格、語学試験、日本語能力試験など

フォースバレー経由では、Excelや所定フォーマットでまとまった情報が来ることがある。一方で、一般案件では履歴書の粒度がまちまちで、名前と本文しかないようなケースもある。AIは「資料を受け取って終わり」ではなく、不足項目を検知して追加質問する役割が重要。

情報管理は、フォーム入力用JSONを正本にしない。ヒアリング、OCR、Excel、添付資料から得た情報は、まず `case_data` に正規化する。そこから入管オンライン申請フォーム用の `application_data` を生成する。

## AI化しやすいところ

AIエージェントで支援しやすい作業は多い。

- メール、添付資料、フォーム回答の取り込み
- OCRによる旅券、履歴書、卒業証明書、成績証明書の読み取り
- 申請フォーム項目へのマッピング
- 不足資料・不足項目の検出
- 追加依頼メールやLINE文面のドラフト
- 学歴・職歴・仕事内容の整合性チェック
- 過去の通過事例から活動内容詳細の草案作成
- オンライン申請システムへの入力補助
- 入力ミス、ローマ字、日付、学校名、契約期間の矛盾チェック

完全自動申請より、最初は「入力補助」「不足確認」「判断アラート」「人の最終確認」を前提にしたほうが現実的。

特に17.2〜17.4の過去出入国歴、21.2〜21.8の在日親族、職歴01〜06のような条件付き・繰り返し項目は、フォーム台帳とマッピングで管理する。案件データ側では、出入国歴や親族を論理的な配列・オブジェクトとして持つ。

## 人が見るべきところ

以下は初期段階では人の確認を残す。

- どの在留資格で申請するか迷う案件
- 技人国と技能、興行、特定技能、高度専門職などの境界案件
- 学歴と職務のつながりが弱い案件
- 活動内容が単純労働に見えそうな案件
- 中国人配偶者系など、直近の運用変更で厳格化している案件
- 過去に不許可・補正があった案件
- 理由書や補足説明で経験値が必要な案件

## 周辺業務

在留資格申請の周辺には、社会保険・雇用保険の加入手続き、助成金、研修、人材紹介、許認可などの派生業務がある。特にシャローム入力支援は、OCRとブラウザ入力自動化で短期にデモ化できる可能性が高いという話があった。
