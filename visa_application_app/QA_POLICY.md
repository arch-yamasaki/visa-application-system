# QA Policy

## 絶対禁止事項

Chrome DevTools MCP、手動QA、Chrome拡張QAのいずれの場合も、申請の最終送信ボタンは絶対に押さないでください。

入力後の作業範囲は、入力結果の確認、画面表示の確認、スクリーンショット取得、コンソール・ネットワークエラー確認までです。送信が必要な場合は、必ず人間が全項目と添付資料を最終確認したうえで、手動で実行してください。

## QAで許可する操作

- ログイン画面または入力画面への移動
- フォーム項目への自動入力
- 入力対象のプレビュー
- 入力結果の目視確認
- スクリーンショット取得
- Console / Network エラー確認
- `case_data` から生成した `application_data` の投入確認

## QAで許可しない操作

- 申請の最終送信
- 「申込む」「送信」「確定」など、行政手続を確定させる操作
- 人間の明示確認なしの添付ファイル送信
- 本番申請番号を発行・更新する可能性がある操作

## データ取扱い

- 実PIIを含むQAは許可する。ただし、対象データとそこから生じる派生物はすべて `restricted test data` として扱う。
- 実案件の `case_data` とデモデータを混ぜないでください。
- Chrome拡張の `application_data.json` は投入用の派生物です。案件正本として編集・保管しないでください。
- 個人情報を含む実案件データを拡張同梱ファイルに入れないでください。
- `confidence=medium` やOCR根拠が弱い項目は、入力前に人間が確認してください。
- 実案件データでQAする場合は、専用Chromeプロファイルを使い、検証終了後に `chrome.storage.local` を消去してください。
- PII表示状態のスクリーンショット、画面録画、DevTools Console、Network export、MCP取得物は機微情報として扱ってください。
- バグ報告やチャット転記では、氏名、住所、電話、メール、旅券番号、申請番号、自由記述を伏せてください。
- QA後は、不要なスクリーンショット、ダウンロード物、ログ、生成物を削除してください。

## restricted test data のリスクと対策

QAでは実データを使ってよいが、漏えい防止のために「何が残るか」を前提に管理する。

| リスク | 何が起きるか | 対策 |
| --- | --- | --- |
| ローカルファイル残存 | 検証用に保存したPDF、ZIP、スクリーンショット、JSONが端末内に残り、別案件作業やバックアップ経由で意図せず閲覧される。 | 保存先を限定し、検証後に不要ファイルを削除する。デスクトップ、Downloads、一時メモ配下へ置きっぱなしにしない。 |
| `chrome.storage.local` 残存 | 前回案件の申請人情報が拡張内に残り、次回QAで誤投入や誤表示が起きる。 | 実データQAは専用Chromeプロファイルのみで行う。案件切替時と終了時に `chrome.storage.local` を消去する。 |
| DevTools console / screenshot / generated JSON 残存 | Console、スクリーンショット、`application_data` 生成物に実PIIが残り、検証メモや添付資料として再配布される。 | Consoleには実値を極力出さない。スクリーンショットと生成JSONは raw と同じ `restricted` 扱いにする。共有前にマスクし、不要な保存物は削除する。 |
| Git / 共有フォルダ混入 | 実PII入りファイルを誤ってコミット、push、同期共有すると、履歴や複製先からの削除が難しい。 | 実PII入りraw、generated、キャプチャをGitや共有同期対象に入れない。コミット前に差分確認する。 |
| bug報告 / チャット転記漏えい | 不具合説明でJSON断片や画面文言をそのまま貼り、会話ログやチケットに機微情報が残る。 | バグ報告は項目名、症状、期待値、再現手順を中心に書く。値が必要な場合はダミー化またはマスクして転記する。 |
